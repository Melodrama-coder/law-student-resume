# 法学生求职简历工作流参考指南

> 与 `law-student-resume` Skill 配套使用。本文件包含模块化提问模板、真实性校验原则、个性化策略以及生成器 JSON 格式说明。仅在执行简历制作时按需读取。

## 1. 模板共性结构

综合三份模板（红圈所/央企法务 PDF、法学生简历 DOCX、精品简历 DOCX）得出：

- **核心必填模块**：个人信息、教育经历、实习/工作经历、专业技能
- **可选补充模块**：科研经历、竞赛经历、项目经历、个人总结
- **输出原则**：一页 A4、简洁大方、无花哨设计、中文优先使用仿宋/微软雅黑、英文统一 Times New Roman。
- **版式模板**：提供 5 种（`classic`/`minimal`/`centered`/`modern`/`compact`），配色默认以 **黑、白、灰、天蓝色** 为主，并支持自定义强调色。

## 2. 信息收集流程（模块化、逐步、口语友好）

> 核心纪律：**一次只问一个模块**；每模块结束**先回显小结让用户确认**，再进入下一模块。绝不在一条消息里索取全部信息——法学生（尤其表达口语化的）极易遗漏。

**标准循环（对每个模块重复）：**
1. 报进度：开头写“第 X / N 步 · 模块名”，让用户知道还剩多少。
2. 提问：给出该模块的问题清单，并明确告诉用户“**你只需用大白话讲你做了什么，措辞我来帮你提炼**”。
3. 深挖：对经历类模块逐条追问“具体做了什么 + 有无数据佐证”（见第 3 节）。
4. 回显小结：用表格或要点复述记录到的内容，问“对不对 / 要补充吗”。
5. 确认后再进入下一模块。

**模块顺序：**
第 0 步（版式+字体+求职方向+盘点有哪些模块）→ 个人信息 → 教育经历 → 实习/工作经历 → 可选模块（科研/竞赛/项目/个人总结，有才问）→ 专业技能 → 真实性确认 → 生成。

### 2.1 口语转书面技巧（针对口语化学生）

- 让用户**自由口述**，不要求其组织语言；你负责转成“动词开头 + 量化 + 结果”的 bullet。
- 一段口述里若混了多段实习/多个项目，**主动帮其拆分**成独立条目，并按倒序排列。
- 提炼后**回读给用户听**，并标注哪些是“你的原话量级/约数”，请其确认未夸大。
- 对含糊词（“很多、大量、经常”）追问具体数字或合理约数；给不出就不写数量。
- 常见口语→书面对照：
  - “整理了一堆档案” → “独立整理子公司工商档案，梳理关联公司股权结构，制表跟踪股权瑕疵问题”
  - “帮律师查资料写报告” → “就 XX 争议完成专项法律检索，梳理数十份判例并按裁判观点分类，形成约万字检索报告”
  - “改过很多合同” → “累计审阅采购、保密等各类合同约 XX 份，标注风险点并提出修改建议”

## 3. 各模块提问模板

### 3.1 个人信息
- 姓名、联系电话、邮箱、出生年月、性别、籍贯
- 政治面貌（中共党员/群众等，如觉得敏感可省略）
- 求职意向（目标岗位：红圈所/央企法务/公检法/其他）
- 是否提供证件照？如提供请给出图片路径。

### 3.2 教育经历
- 学校、学院、专业、学位、时间、地点
- GPA、专业排名（百分比或名次/总人数）
- 核心课程（3-6 门与求职方向最相关的）
- 荣誉奖项（奖学金、荣誉称号、级别）
- **追问**：排名有没有官方证明？GPA 是 4.0 还是 5.0 制？奖学金是否确有其事？

### 3.3 实习/工作经历（每个经历分别问）
- 单位全称、部门/团队、岗位、城市、起止时间
- 你在这个岗位上的主要职责是什么？
- 你具体参与了哪些案件/项目/任务？
- 你使用了哪些工具/数据库/方法？（如威科先行、北大法宝、尽调底稿、合同审查清单）
- **追问**：有没有可量化的数据？例如：
  - 起草/审查合同、法律意见书、尽调报告 XX 份
  - 参与 XX 起诉讼/仲裁案件，涉案金额 XX 万元
  - 完成网络核查、访谈、底稿整理 XX 项
  - 检索法律法规/案例 XX 条，形成 XX 万字报告
  - 协助团队服务 XX 家常法客户
- 要求用 **STAR** 或 **PAR**（Problem-Action-Result）组织 bullet。

### 3.4 科研经历（可选）
- 项目名称、项目来源或级别、担任角色、时间
- 研究主题、研究方法、你的具体贡献
- 成果形式：论文、研究报告、获奖、发表字数
- **追问**：项目是否真实立项？你的贡献可量化吗？（检索量、字数、代码/数据量等）

### 3.5 竞赛经历（可选）
- 竞赛名称、级别、获奖/名次、担任角色、时间
- 你具体做了什么？（书状撰写、庭辩、检索、可视化）
- **追问**：团队多少人？你在团队中的排名？是否有官方证书或截图？

### 3.6 项目经历（可选）
- 项目/课题名称、来源、角色、时间
- 项目目标、你承担的任务、具体产出
- **追问**：与科研经历是否重复？若重复保留最有含金量的一项。

### 3.7 专业技能
- 法律职业资格：是否通过法考？客观题/主观题分数？
- 语言：英语四六级、雅思/托福分数；是否可英文起草/审阅法律文件？
- 办公与检索：Office、Python、威科先行、北大法宝、无讼、Alpha、企查查、WIPO/WTO 数据库等
- 其他证书：计算机等级、实习证明、结业证书等

### 3.8 个人总结（可选，字数控制在 3-4 行）
- 教育背景与目标岗位的匹配点
- 实务能力亮点
- 软技能/职业素养
- 避免空泛形容词，用“X 段实习/通过法考/英语 X 分”等具体信息支撑。

## 4. 真实性校验（必须）

在整理内容时，必须向用户明确提出：

- “以上信息是否均为你的真实经历？数据是否可核查？”
- 对模糊数字（如“参与了多个项目”“负责大量文书”）追问具体数字或范围。
- 如果用户无法提供数据，宁可不写数量，也不要编造“处理 100 份合同”这类数据。
- 如发现内容与模板示例重复或明显照搬，提醒用户替换为真实经历。
- 禁止为美化简历而虚构单位、岗位、获奖或法考分数。

## 5. 个性化策略

- **实习经历丰富**：把实习作为简历主体，每个经历配 2-3 条量化 bullet，科研/竞赛可弱化或合并。
- **实习经历较少**：强化教育背景（GPA、排名、课程、奖学金）、科研、竞赛、模拟法庭、法律援助、课程项目；强调可迁移技能（检索、写作、演讲、数据分析）。
- **非传统路径**：突出复合背景（本科非法律 + 法律硕士/双学位）、语言优势、实习转向说明。
- **应聘央企法务**：更强调合规、合同审查、风险管理、跨部门沟通经验；字体推荐仿宋，风格稳重。
- **应聘红圈所**：更强调交易/诉讼/非诉项目、尽职调查、英文能力、高强度工作表现；字体推荐微软雅黑，风格现代专业。

## 6. 精简到一页的取舍原则

- 每个 bullet 控制在一行半以内，避免长篇大论。
- 删除与求职目标无关的经历或泛泛描述。
- 合并同类低权重经历（如只写“在某律所参与 2 起尽调”可合并为一条）。
- 保留含金量最高、与目标岗位最相关的内容。
- 如果用户内容实在过多，提示其删减至一页优先，或接受更小字号（生成器会自动适配）。

## 6.1 一页裁剪决策树（内容溢出时）

当脚本提示“超出一页”时，按以下优先级逐级精简（脚本也会自动给出**『保守方案 / 激进方案』两份建议**供选择）：

1. **优先删（不损核心）**
   - 与求职方向无关的社团 / 兼职 / 学生干部经历。
   - 非核心课程（仅保留与方向最相关的 3–6 门）。
   - 空泛的“自我评价 / 个人总结”套话。
2. **其次合并（同类压缩）**
   - 3 条相似 bullet 合并为 1 条概括，如“累计审阅合同约 400 份，覆盖采购 / 保密 / 服务类”。
   - 多个小项目合并为“参与 X 起 XX 类型项目”。
3. **最后降级（动词瘦身）**
   - “独立完成” → “参与 / 协助”（更稳妥也更省篇幅）。
   - 长 bullet 压到一行半以内，删修饰词。
4. **仍超页**：直接删可选模块（科研 / 竞赛 / 项目 / 总结），只留 教育 + 实习 + 技能。

判定口诀：**无关先砍 → 同类合并 → 动词降级 → 可选模块取舍**。

## 7. 生成器 JSON 格式

`scripts/generate_resume.py` 接收如下 JSON 结构（示例见 `assets/sample_resume.json`）：

```json
{
  "meta": { "font": "yahei", "template": "classic", "accent": null, "photo": null, "language": "zh" },
  "personal": {
    "name": "姓名",
    "phone": "+86 138-0000-0000",
    "email": "name@example.com",
    "gender_age": "男 | 2000.05",
    "native_place": "籍贯：XX",
    "political": "中共党员",
    "job_intention": "求职意向：红圈所/央企法务"
  },
  "education": [...],
  "internship": [...],
  "skills": {...},
  "summary": "...",
  "en": { "personal": {...}, "education": [...], "internship": [...], ... }
}
```

- `meta.language` 可选：`zh`(中文)、`en`(英文)、`both`(中英各一份)。英文版使用 `data.en` 块覆盖对应内容；若 `en` 块缺失，章节标题与联系标签已自动英译，正文自由文本回退中文并提示补充（见 §13）。
- `meta.font` 可选：`fang` / `yahei` / `song` / `kaiti`。`meta.template` 可选：`classic` / `minimal` / `centered` / `modern` / `compact`。
- `meta.accent` 为可选强调色覆盖：颜色名（如 `skyblue`、`black`、`gray`、`navy`、`teal`）或十六进制 `#RRGGBB`；不填则使用模板默认色。命令行 `--accent` 可覆盖 JSON。
- `meta.photo` 为可选证件照路径。
- `meta.style` 可选语言风格：`aggressive`（红圈所·突出主导/结果）/ `moderate`（综合·平衡团队与个人）/ `conservative`（体制内·强调严谨/合规）。命令行 `--style` 可覆盖。该风格指导“口语→书面”的改写用词（见 §11）。
- `meta.versions` / 命令行 `--versions` 支持**一稿多投**：填 `law` / `legal` / `civil` 的任意组合（如 `law,legal`）。脚本按版本重排模块顺序并生成多份文件（见 §12）。
- `versions_data`（可选）：按版本 key 给专属内容覆盖，实现不同版本的 bullet 权重差异。例：
  ```json
  "versions_data": {
    "law":   { "internship": [ "突出案件、检索能力、英文的 bullet …" ] },
    "legal": { "internship": [ "突出合同管理、合规意识、风险排查的 bullet …" ] }
  }
  ```
- 命令行 `--font` / `--template` / `--style` / `--versions` 都会覆盖 JSON 中的 `meta` 设置。

> 更多“口语原文 → 简历 bullet”的转化范例与各领域量化指标，见 **`references/examples.md`**（案例库 / 话术库）。
- 所有经历数组只保留要展示的条目，空数组或空字段会自动跳过。英文版若使用 `meta.language: en`，内容会从 `data.en` 中读取；缺失时回退中文（脚本会提示建议补充）。
- 英文统一使用 Times New Roman，无需在 JSON 中额外设置。
- 专业技能（`skills`）每段以「标签：内容」书写，冒号前的**标签会自动加粗**（全部模板一致）；`extra` 中的自由行若含冒号也同样处理。

## 8. 调用方式

### 8.1 常规生成（Word + PDF）

```bash
python scripts/generate_resume.py \
  --data assets/sample_resume.json \
  --out ./output \
  --name 张明_简历 \
  [--font fang] [--template minimal] [--accent navy] [--style moderate]
```

输出：`张明_简历.docx` + `张明_简历.pdf`。脚本会自动从 **12pt 起**逐档尝试字号，选择**能单页容纳的最大字号**；内容少则字大饱满，内容多则适当缩小。若内容实在过多仍超出一页，会提示用户精简。

### 8.2 英文版 / 中英双版

```bash
# 仅英文
python scripts/generate_resume.py --data resume.json --out ./output --name 张明_简历 --language en

# 中英各一份
python scripts/generate_resume.py --data resume.json --out ./output --name 张明_简历 --language both
```

- 英文内容需在 JSON 顶层加 `en` 块，结构同顶层各模块（见 §7）。章节标题与联系标签已自动英译，不确定的法律术语参考 `references/legal_terms.md`。
- 若 `--language en` 但 `data.en` 缺失，脚本会回退中文并提示补充英文 bullet。

### 8.3 LaTeX 导出（隐藏功能）

```bash
python scripts/generate_resume.py --data resume.json --out ./output --name 张明_简历 --format latex
```

- 生成 `.tex` 文件，需本地安装 `ctex`（中文）或 `times`（英文）宏包，用 `xelatex` 编译。
- 适合对排版精度要求极高的外所/学术岗位。

### 8.4 微调模式（patch_resume.py）

生成后若只想改某个字段，不必重走流程：

```bash
# 修改邮箱 + 替换第 1 段实习的第 2 条 bullet
python scripts/patch_resume.py \
  --base output/张明_简历.json \
  --set "personal.email=new@example.com" \
  --set "internship[0].bullets[1]=新的 bullet 文本" \
  --out output/张明_简历_v2.json

# 在第 1 段实习末尾追加 bullet
python scripts/patch_resume.py \
  --base output/张明_简历.json \
  --add "internship[0].bullets=\"新增 bullet\"" \
  --out output/张明_简历_v2.json

# 删除第 2 段实习
python scripts/patch_resume.py \
  --base output/张明_简历.json \
  --del "internship[1]" \
  --out output/张明_简历_v2.json

# 合并一个 JSON 片段
python scripts/patch_resume.py \
  --base output/张明_简历.json \
  --patch resume_fragment.json \
  --out output/张明_简历_v2.json
```

改完后只需用新 JSON 重新运行 `generate_resume.py` 即可。

### 8.5 断点续传：汇总 JSON 与 merge_resume.py

每完成一个模块，向用户展示「当前汇总 JSON」代码块并提醒复制保存。下次粘贴即可继续，或用：

```bash
python scripts/merge_resume.py \
  --base 旧简历.json \
  --patch 新补充模块.json \
  --out 合并后.json
```

### 8.6 岗位匹配度雷达图（match_score.py）

```bash
python scripts/match_score.py \
  --data resume.json \
  --target law_firm \
  --out match_score.png
```

- 可选 `--target`：`law_firm`（红圈所/精品所）、`corporate_legal`（央企/企业法务）、`foreign_firm`（外所/国际仲裁）。
- 输出 6 维雷达图（学历 / 实习 / 职业资格 / 语言 / 工具 / 竞赛科研）+ 文本诊断与提升建议。
- 若提示缺少 `matplotlib`，请运行 `pip install matplotlib`。其他依赖同理，脚本会友好提示。

## 9. 版式模板、配色、预览与选择建议

| 模板 | 默认风格 | 默认强调色 | 适配求职方向 |
|---|---|---|---|
| `classic` | 经典，标题带英文副标题+分隔线 | 天蓝色 `skyblue` | 红圈所 / 综合投递（默认） |
| `minimal` | 极简黑白，无彩色、细灰线 | 黑色/灰色 | 央企法务 / 公检法（稳重） |
| `centered` | 居中庄重，姓名与标题居中 | 深灰色 | 体制内正式风（推荐配仿宋） |
| `modern` | 现代，姓名色块+方块标题 | 天蓝色 `skyblue` | 新锐律所 / 互联网法务 |
| `compact` | 紧凑填满，黑底白字标题栏+编号 bullet+右上证件照 | 黑色 `black` | 希望一页饱满、信息密度高（参考 offer6 风格） |

- 默认配色遵循 **黑、白、灰、天蓝色** 的专业、稳重原则；避免过度花哨。
- 若用户想要其他颜色，可在 JSON 的 `meta.accent` 或命令行 `--accent` 指定：颜色名（如 `navy`、`teal`、`burgundy`）或十六进制（如 `#C0392B`）。
- 用户犹豫时，可用同一份 JSON 循环生成多种模板+配色供对比，或查看 `assets/templates/preview.html`（本地打开即可横向对比 5 模板）。
- **渲染对比图**（自检或给用户看，需 pymupdf + pillow）示例：
  ```bash
  python scripts/render_thumbnails.py
  # 生成 assets/templates/classic.png ... compact.png
  ```
  再用 `assets/templates/preview.html` 打开浏览。
- 模板与配色只改**视觉风格**（版式、强调色、页眉/标题样式），不改内容与一页约束。

## 10. 无实习 / 无工作经历的替代方案

若用户完全没有实习或工作经历，`generate_resume.py` 的校验会提醒，并建议用以下**替代素材**避免简历空洞（均须真实、可核查）：

1. **模拟法庭 / 辩论赛**：担任角色（辩方 / 原告代理人 / 书状主笔）、具体职责（法律检索、书状撰写、庭辩）、成果（获奖 / 名次 / 团队人数）。
2. **课程 / 科研课题**：课题名称、你的角色、具体贡献（文献检索、数据整理、报告撰写）、产出（报告字数 / 是否被收录）。
3. **法律援助 / 社团法务 / 志愿服务**：提供法律咨询、起草文书、普法宣讲等可量化动作。
4. **可迁移技能兜底**：法律检索、文书写作、口头表达、基础数据分析（如辅修会计、会用 Excel/Python）。

> 原则：宁可内容少而真实，也不要编造实习。替代素材按“动词 + 动作 + 量化结果”同样提炼。

## 13. 依赖与友好报错

运行脚本前请确保已安装依赖：

```bash
pip install python-docx pypdf pymupdf pywin32 pillow matplotlib
```

- 如果脚本提示缺少 `python-docx`、`pypdf`、`pymupdf`、`pywin32`、`pillow` 或 `matplotlib`，会给出明确的 `pip install` 命令，而不是抛出 Python 堆栈。
- PDF 生成依赖 Microsoft Word（Windows）或 LibreOffice（Linux/macOS），未安装时脚本也会友好提示。
- 所有生成的 PDF 已自动写入元数据：标题、作者、生成时间（`GeneratedAt`），便于版本管理。

## 11. 语言风格引擎（口语 → 书面的三种措辞）

同一段经历，按求职方向切换措辞**风格**（命令行 `--style` 或 `meta.style`）。脚本本身不重写文字，风格由你在收集/提炼时套用以下动词库：

| 风格 | 适用 | 动词库（突出点） | 改写倾向 |
|---|---|---|---|
| `aggressive` | 红圈所 / 顶级外所 | 主导、独立承办、牵头、创造性解决、突破 | 强调**个人主导、独立、结果、金额** |
| `moderate` | 综合律所 / 一般企业 | 参与、协助、独立完成、负责 | **团队与个人平衡** |
| `conservative` | 央企法务 / 公检法 / 体制内 | 协助落实、严格依照、规范、稳妥推进、服从统筹 | 强调**严谨、合规、稳妥、服从大局** |

**示例（同一段实习的三种写法）：**
- aggressive：「**主导**某 IPO 项目尽职调查，独立核查 30 余家关联公司，发现 5 处股权瑕疵，**牵头**撰写尽调报告初稿。」
- moderate：「**参与**某 IPO 项目尽职调查，核查关联公司股权结构，审阅合同约 400 份，**独立完成**尽调报告初稿。」
- conservative：「**协助**团队开展 IPO 尽职调查工作，严格依照尽调清单核查关联公司，**稳妥**整理并归档工作底稿。」

## 12. 多版本简历（一稿多投）

法学生常同时投律所 + 央企法务，但侧重点不同。生成器支持一次性产出多个版本：

- **触发**：命令行 `--versions law,legal`（可任意组合 `law` / `legal` / `civil`），或 JSON 中 `meta.versions: ["law","legal"]`。
- **版本差异（模块顺序自动重排）**：
  - `law`（律所版）：实习 → 工作 → 教育 → 技能 …（**突出案件、检索、英文**）
  - `legal`（法务版）：工作 → 实习 → 教育 → 技能 …（**突出合同、合规、风险**）
  - `civil`（体制版）：教育 → 竞赛 → 科研 → 实习 …（**突出学历、竞赛、稳健**）
- **输出**：文件名自动加后缀 `_law` / `_legal` / `_civil`，抬头求职意向后标注版本名（如「红圈所（律所版）」）。
- **bullet 权重差异**：在 JSON 顶层加 `versions_data`，按版本 key 覆盖对应模块的 bullet，使不同版本强调不同内容（见 §7 示例）。

```bash
python scripts/generate_resume.py --data resume.json --out ./output \
  --name 张明_简历 --versions law,legal
# 生成：张明_简历_law.docx/pdf 与 张明_简历_legal.docx/pdf
```
